# Agent Review — VSCode Extension PRD

## Overview

Agent Review is a VSCode extension that displays AI-generated code review comments inline, using VSCode's native Comments API. It works alongside the GitHub Pull Requests extension so both AI-generated and human-generated review comments appear side-by-side in the editor.

## Current State

The extension is functional with these features:
- Reads review JSON files from `.reviews/` directory in the workspace
- Displays comments inline at correct file:line locations
- Supports git submodule path resolution via `.gitmodules` parsing
- Severity-based icons and labels (blocking, important, suggestion, nit, praise, learning)
- Author information from JSON (per-comment or review-level default)
- File watching with auto-refresh on JSON changes
- Interactive actions: edit comment body, change severity, delete comment (all persist to JSON)
- Commands: Load Review File, Reload Comments, Clear All Comments

## Remaining Tasks

### 1. Fix edit/save flow
The edit flow (pencil icon -> edit text -> save) needs verification that it correctly persists the updated body to the JSON file. The save/cancel buttons were moved to `comments/comment/context` and a `isSaving` flag was added to suppress watcher reloads. Needs end-to-end testing and any fixes required.

### 2. Add GitHub posting command
Add a command `agentReview.postToGitHub` that posts all comments from the current review to the GitHub PR as a single review. Should use the `gh` CLI or GitHub REST API. The review body should be the summary overview, the event should map from the verdict (APPROVE/REQUEST_CHANGES/COMMENT), and each comment should be attached with its path, line, side, and body.

### 3. Package as installable VSIX
Add `@vscode/vsce` as a dev dependency with a `package` script. Document the install flow. Test that the packaged extension activates correctly in a fresh VSCode window without the Extension Development Host.

### 4. Initial git commit
Create the first commit for the agent-reviews repository with all current files. Use conventional commit format.

### 5. Update code-review-excellence skill output path
The Claude Code skill `/code-review-excellence` currently saves review files to `/tmp/`. Update it (or document how to update it) to save directly to the target project's `.reviews/` directory so the extension picks them up automatically.

### 6. Add status bar item
Show a status bar item when a review is loaded indicating the PR number, verdict, and comment count. Clicking it should open the review summary or reload comments.

## Technical Context

- Extension location: ~/Documents/smallcase/agent-reviews/
- Built with TypeScript, compiled via tsc (no bundler)
- Uses VSCode Comments API (CommentController, CommentThread, Comment)
- Review JSON format includes: pr metadata, author, summary (verdict/overview/strengths), and comments array (path/line/side/severity/body/timestamp/author)
- Submodule resolution parses .gitmodules to map repo slugs to local paths
